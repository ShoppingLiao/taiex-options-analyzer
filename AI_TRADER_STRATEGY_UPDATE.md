## 📋 更新概述

將 AI 交易員的投資策略從「一般性結算日操作」調整為**專注於週三/週五組合單當沖策略**。

### 核心理念

所有的單日報告都是為了讓 AI 交易員能提早準備好週三和週五的組合單操作。

## 🎯 新策略核心

### AI 交易員定位

- **專職角色**: 結算日組合單交易員
- **操作頻率**: 僅週三和週五（結算日）
- **交易方式**: 選擇權組合單當沖
- **結算機制**: 13:30 系統自動結算

### 投資理念

```
週一 → 觀察市場、分析數據
週二 → 觀察市場、分析數據
週三 → 組合單當沖交易（第一次結算）
週四 → 觀察市場、分析數據
週五 → 組合單當沖交易（第二次結算）
```

**關鍵概念**: 平日的所有報告都是為了週三/週五做準備！

## 💼 組合單策略

### 根據趨勢選擇組合

#### 1. 多方趨勢（Bullish）→ Bull Call Spread

```
買進 Call (較低履約價) - 付權利金
賣出 Call (較高履約價) - 收權利金
───────────────────────────────
淨支出: 權利金差額（有限）
最大獲利: 履約價差 - 淨權利金
最大虧損: 淨權利金
```

**適用情境**: 趨勢強度 ≥ 3 顆星，看好突破上緣

#### 2. 空方趨勢（Bearish）→ Bear Put Spread

```
買進 Put (較高履約價) - 付權利金
賣出 Put (較低履約價) - 收權利金
───────────────────────────────
淨支出: 權利金差額（有限）
最大獲利: 履約價差 - 淨權利金
最大虧損: 淨權利金
```

**適用情境**: 趨勢強度 ≥ 3 顆星，預期跌破下緣

#### 3. 趨勢不明（Neutral）→ Iron Condor

```
Call Side:
  賣出 Call (價外) - 收權利金
  買進 Call (更價外) - 付權利金

Put Side:
  賣出 Put (價外) - 收權利金
  買進 Put (更價外) - 付權利金
───────────────────────────────
淨收入: 權利金總和（收入型策略）
最大獲利: 淨權利金
最大虧損: 單邊價差 - 淨權利金
```

**適用情境**: 趨勢強度 ≤ 2 顆星，預期區間震盪

## ⏰ 結算日操作流程

### D-1 (前一天晚上)

- ✅ 複習本週所有單日報告
- ✅ 確認趨勢方向和強度
- ✅ 規劃組合單策略
- ✅ 計算預期獲利/虧損
- ✅ 準備券商組合單下單功能

### 08:00-08:45 (開盤前)

- 檢查隔夜美股和國際盤
- 確認組合單履約價選擇
- 檢查流動性（成交量 > 100）
- 準備下單介面

### 08:45-09:30 (早盤觀察)

- **不進場**，僅觀察
- 確認開盤方向是否符合預期
- 觀察大單流向
- 評估是否需調整策略

### 09:30-10:30 (第一批建倉)

- 建立 50% 部位
- 使用「組合單下單」功能
- 設定獲利 60% / 虧損 50% 條件單
- 記錄建倉價格

### 10:30-11:30 (觀察與加碼)

- 評估第一批損益
- 獲利 20-30% 可考慮加碼剩餘 50%
- 虧損 > 30% 準備停損

### 11:30-12:30 (部位管理)

- **關鍵決策時段**
- 獲利 60% → 平倉落袋
- 虧損 50% → 停損出場
- 損益 ±20% → 續抱到結算

### 12:30-13:15 (結算前決策)

- 不再調整部位
- 評估價格與預測的距離
- 價格大幅偏離 → 緊急平倉

### 13:15-13:30 (結算倒數)

- **不要調整組合**
- 避免單腿風險
- 讓系統自動結算

### 13:30 之後 (結算後)

- 查看最終損益
- 記錄操作心得
- 更新學習系統
- 準備下次交易

## 🛡️ 風險控管

### 部位管理

- 單次組合單 ≤ 總資金 30%
- 分 2-3 批建倉
- 嚴守獲利 60% / 虧損 50% 紀律

### 履約價選擇

- 選擇成交量 > 100 的履約價
- 價內一檔、價平、價外一檔流動性最好
- 避免太價外的履約價

### 組合單執行

- 必須使用「組合單下單」功能
- 避免單腿成交風險
- 檢查每個履約價的流動性

### 時間管理

- 最後 30 分鐘不調整部位
- 12:30 前決定是否持有到結算
- 避免最後時刻的流動性風險

## 💡 組合單優勢

### 1. 風險有限

- 建倉時就知道最大虧損
- 不會因市場波動無限擴大
- 心理壓力較小

### 2. 時間優勢

- 結算日時間價值快速歸零
- Theta Decay 站在我們這邊
- 只要方向對，獲利隨時間增加

### 3. 自動結算

- 13:30 系統自動平倉
- 不用擔心忘記出場
- 避免隔夜留倉風險

### 4. 高勝率

- 掌握莊家結算價位規律
- Max Pain 是重要參考指標
- 結合每日報告的趨勢分析

### 5. 明確收益

- 最大獲利和虧損都清楚
- 風險收益比易於評估
- 有利於資金管理

## 🎯 策略調整的意義

### 為什麼專注結算日？

1. **時間可控**: 每週只有兩天交易，其他時間研究和準備
2. **風險明確**: 組合單最大虧損在建倉時就確定
3. **自動結算**: 13:30 系統平倉，避免人為疏失
4. **高勝率**: 結合每日報告分析，提高判斷準確度
5. **可複製**: 固定的流程和紀律，容易持續執行

### 每日報告的新意義

所有週一到週五的單日報告，都是為了：

- 觀察趨勢方向的變化
- 追蹤 OI 分布的演變
- 分析 P/C Ratio 的訊號
- 預測結算價可能區間
- 為週三/週五做準備

**核心概念**: 平日累積資訊 → 結算日精準出擊

## 📝 範例分析

### 多方情境（趨勢強度 4 顆星）

**預測**:

- 結算區間: 29,800 - 30,200
- Max Pain: 29,900
- 當前價格: 29,850

**策略**: Bull Call Spread

```
買進 Call 29,800 (價內)
賣出 Call 30,000 (價外)
───────────────────
履約價差: 200 點
假設淨權利金: 80 點
最大獲利: 120 點 (當結算價 ≥ 30,000)
最大虧損: 80 點 (當結算價 ≤ 29,800)
風險收益比: 1:1.5
```

**判斷理由**:

- 多方趨勢明確
- 預測上緣 30,200 > 賣出價 30,000
- Max Pain 在買進價附近，有支撐
- 風險收益比合理
